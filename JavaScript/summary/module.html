모듈

코드의 재활용성을 높이고, 유지보수를 쉽게할 수 있는 다양한 기법이 사용됨
그 과정 중 하나가 코드를 여러 개의 파일로 분리하는 것 : 모듈화

  • 자주 사용되는 코드를 별도의 파일로 만들어서 필요할 때마다 재활용 가능
  • 코드를 개선하면 이를 사용하고 있는 모든 애플리케이션의 동작 개선 가능
  • 코드 수정 시에 필요한 로직을 빠르게 찾을 수 있음
  • 필요한 로직만을 로드하여 메모리 낭비를 줄임
  • 한번 다운로드된 모듈은 웹브라우저에 의해서 저장되기 때문에 동일한 로직을 로드할 때 시간과 네트워크 트래픽 절약 가능

자바스크립트에서는 모듈이라는 개념이 분명하게 존재되진 않지만, 구동되는 호스트 환경에 따라서 서로 다른 모듈화 방법이 제공되고 있음

호스트 환경이란?

자바스크립트가 구동되는 환경을 의미

라이브러리

모듈이 프로그램을 구성하는 작은 부품으로서의 로직을 의미한다면 라이브러리는 자주 사용되는 로직을 재사용하기 편리하도록 정리한 일련의 코드들의 집합을 의미

자바스크립트에서는 jQuery, React, Vue.js, Angular 등이 있음

UI

User Interface : 사용자를 마주하는 접점, 사용자가 제품/서비스와 상호작용 할 수 있도록 만들어진 매개체

API

  • Application Programming Interface : 프로그램이 동작하는 환경을 제어하기 위해서 환경에서 제공되는 조작 장치
  • 조작 장치는 프로그래밍 언어를 통해서 조작이 가능함
  • 자바스크립트의 API는 크게 자체 API와 호스트 환경의 API로 구분
